# Contributing to Kaffenol

Kaffenol is not accepting implementation work yet. During the foundation phase, contributions should clarify product intent, architectural boundaries, decisions, and build conventions.

Before proposing a change:

1. Read `AGENTS.md` and `docs/INDEX.md`.
2. Check relevant architecture decision records.
3. Keep one change focused on one reviewable outcome.
4. Use `docs/plans/TEMPLATE.md` for substantial or cross-module work.
5. Run `./scripts/check.sh` and report any checks that could not run.

Do not add generated files, credentials, signing material, large binary fixtures, or new dependencies without documenting their purpose.
